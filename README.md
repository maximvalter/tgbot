# Телеграм-бот для изучения английского языка

Эта инструкция описывает, как подготовить и запустить Telegram-бота для изучения английских слов. Сначала, скопируйте папку проекта на свой компьютер и откройте терминал в её корне. Установите зависимость библиотекой `pip install pytelegrambotapi`. 

Получите токен у [@BotFather](https://t.me/BotFather) и сохраните его в файле `config.py` рядом с `main.py`, записав в нём строку `TELEGRAM_BOT_TOKEN = "ВАШ_ТОКЕН"`. Убедитесь, что в корне проекта есть файл `.gitignore` со строками `.venv/`, `config.py`, чтобы эти директории и файл токена не попали в репозиторий, если возникнет необходимость его публиковать на гитхабе.  

После этого запустите скрипт командой `python main.py` или в интерфейсе вашего IDE. При первом старте автоматически создаются таблицы `users`, `words` и `user_solved` в базе `english_bot.db`, а в таблицу `words` загружаются 10 глобальных слов. В Telegram найдите своего бота по username и нажмите «Start» или отправьте `/start`. Бот пришлёт приветственное сообщение и покажет первую карточку: русское слово и четыре варианта английского перевода. При выборе правильного варианта бот ответит «Правильно! 🎉» и предложит новую карточку. Если ответ неверен, бот напишет «Неправильно. Надо: 🇷🇺 <правильный перевод>» и позволит попробовать снова. Кнопка «Дальше ⏭» пропускает текущее слово и выдаёт новое.  

Чтобы добавить своё слово, нажмите кнопку «Добавить слово ➕», введите сначала английское слово, затем его перевод, после чего бот подтвердит добавление сообщением «Слово 'X' → 'Y' добавлено. Всего слов: N», где N — сумма глобальных и ваших личных слов. Для удаления нажмите «Удалить слово 🔙», бот выведет список ваших добавленных слов и попросит ввести точное название для удаления, затем подтвердит или сообщит, что слово не найдено. Все данные (пользователи, слова и прогресс) сохраняются в базе данных и не теряются при перезапуске скрипта. Если потребуется начать с чистого листа, просто остановите бота и удалите файл `english_bot.db`; при следующем запуске база создастся заново с исходным набором слов.

## Схема базы данных

### Таблица `users`
| Поле         | Тип       | Ключи                              | Описание                                  |
|--------------|-----------|------------------------------------|-------------------------------------------|
| `id`         | INTEGER   | PRIMARY KEY AUTOINCREMENT          | Внутренний уникальный ID пользователя     |
| `telegram_id`| INTEGER   | UNIQUE                             | ID пользователя в Telegram                |
| `username`   | TEXT      |                                    | Никнейм пользователя (может быть NULL)    |
| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP          | Дата и время первой регистрации пользователя |

---

### Таблица `words`
| Поле         | Тип       | Ключи                              | Описание                                         |
|--------------|-----------|------------------------------------|--------------------------------------------------|
| `id`         | INTEGER   | PRIMARY KEY AUTOINCREMENT          | Внутренний уникальный ID слова                   |
| `target`     | TEXT      | NOT NULL                           | Слово на английском                              |
| `translate`  | TEXT      | NOT NULL                           | Перевод слова на русском                         |
| `added_by`   | INTEGER   | FOREIGN KEY → `users(id)` (NULL OK) | ID того, кто добавил слово; `NULL` = глобальное  |

---

### Таблица `user_solved`
| Поле         | Тип       | Ключи                              | Описание                                         |
|--------------|-----------|------------------------------------|--------------------------------------------------|
| `id`         | INTEGER   | PRIMARY KEY AUTOINCREMENT          | Внутренний уникальный ID записи                  |
| `user_id`    | INTEGER   | FOREIGN KEY → `users(id)`          | Пользователь, который отгадал слово              |
| `word_id`    | INTEGER   | FOREIGN KEY → `words(id)`          | Слово, которое было отгадано                     |
| `solved_at`  | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP          | Дата и время, когда слово было успешно отгадано  |
